# Contributing to Charliebox

Thanks for your interest in contributing.

## Getting Started

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Run tests
5. Submit a pull request

## Development Setup

```bash
git clone https://github.com/charlieboxhq/charliebox.git
cd charliebox
```

## Pull Requests

- Keep PRs small and focused
- Write descriptive commit messages (why, not what)
- Include tests for new functionality
- Update relevant documentation

## Reporting Issues

Use GitHub Issues. Include:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Environment details (OS, Python/Node version)

## Code Style

- Python: ruff for formatting and linting
- JavaScript: standard style
- Commit messages: imperative mood, present tense
